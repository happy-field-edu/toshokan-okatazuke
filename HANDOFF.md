## 4. アプリの主な仕様（これまでの要件の到達点）
- 2フェーズ：①ラベルを見てマップで正しい棚をタップ ②同じ棚で正しい順（分類番号／著者あいうえお／巻数）の隙間に入れる。
- 実在の人気本データから**毎回3問**をランダム生成（0〜9類＋えほんE の全分類）。
- 背景は**実際の図書館マップ（map.jpg）**。その上に当たり判定エリア（コード内 `AREAS`、%指定）を重ねる。9類は複数箇所タップ可。
- ラベルは**図書館の実物どおり**：左＝大きい分類1桁／右3段＝①残りの分類番号(基本1桁・たまに2桁)②著者の頭文字(ひらがな1字)③巻数。
- 分類は基本2桁、ところどころ3桁混在（例 596・489）。並べ替えは**NDC順（1桁ずつ比較）**。
- ユニバーサルデザイン：文節で改行／ドラッグ中の本を指の上80pxにずらす／空きスペースが点滅して誘導／ドラッグが固まらない（pointercancel対応）。
- スタート画面＝表紙イラスト（ロイロのWebカードでサムネ表示→虫眼鏡で開ける）。
- ヘッダーに「かえした本」累計（アプリを開いてからの合計、リロードまで保持）。

## 5. アプリの更新手順（必要なときだけ）
1. `okatazuke-master.html` を編集（データは `BOOKS`／当たり判定は `AREAS`／地図は `map.jpg`）。
2. `cp okatazuke-master.html github-app/index.html`（必要なら `map.jpg` もコピー）。
3. `cd github-app && git add -A && git commit -m "..." && git push`
4. 反映に1〜2分。ブラウザは再読み込み。

## 6. 指導案の続け方（いまのメイン作業）
- 提出物：`指導案_としょかんおかたづけ.docx`
- 直し方A（小さな文言）：Wordで直接編集。
- 直し方B（構成・吹き出し等／推奨）：`指導案ジェネレーター.js` を編集 →
  ```
  npm install docx      # 初回のみ
  node 指導案ジェネレーター.js   # docxを再生成
  ```
- 大研様式の必須点：**授業改善の視点を「吹き出し（callout関数）」で、どの場面かを示す**。
